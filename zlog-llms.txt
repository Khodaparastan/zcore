# zsh-log (zlog)

> A high-performance, feature-rich logging library for Zsh.
> Single file (`zlog`), no dependencies, pure Zsh 5.8+.

## Overview

`zlog` replaces ad-hoc `echo` logging in Zsh scripts with structured, leveled, multi-destination logging. Key features:

- **Leveled output** — `error`, `warn`, `info`, `debug` with independent console and file thresholds
- **Structured fields** — append `key value` pairs to any log line, in text or JSON
- **File rotation** — size-based, multi-process safe, configurable retention
- **Context loggers** — named loggers that auto-attach fields to every message
- **Benchmarking** — wrap commands or blocks to get formatted duration log lines
- **Rate limiting & log-once** — prevent log floods in tight loops
- **Async mode** — non-blocking writes via a background worker (experimental)
- **Performance mode** — hot-swap to a fast-path engine (~3–4× faster) for hot loops
- **No dependencies** — pure Zsh 5.8+, single file, safe to re-source

## Requirements

- Zsh 5.8+ (uses `$EPOCHREALTIME`, associative arrays, `zsh/datetime`)
- `stat` — for file rotation size checks (GNU or BSD; auto-detected)
- `mktemp` — for async FIFO (standard on all platforms)

## Install & Source

```zsh
curl -fsSL https://raw.githubusercontent.com/khodaparastan/zsh-log/main/zlog -o zlog
source /path/to/zlog
```

No package manager, no build step.

## Conventions

| Convention | Meaning |
|---|---|
| `REPLY` | Functions that return a value set `$REPLY` instead of using subshells |
| `[key val ...]` | Optional trailing key-value pairs appended to the log line |
| Returns `0` | Success; non-zero on validation failure |
| Level names | Case-insensitive: `error`, `warn`/`warning`, `info`, `debug` or `0`–`3` |

## Output Formats

**Text format** (default):
```
2026-03-01 12:30:45 [INFO ] (1234) Server started | port=8080 env=production
```

**JSON format** (`z::log::set_format json`):
```json
{"timestamp":"2026-03-01T12:30:45.123+00:00","level":"INFO","message":"Server started","hostname":"myhost","pid":1234,"port":"8080","env":"production"}
```

## API Reference

### 1. Quick Start

#### `z::log::setup <file> [level] [format]`
One-call configuration helper. Sets file, level, and format in a single invocation.
- `file` — log file path; use `"-"` for console-only
- `level` — `error`/`warn`/`info`/`debug` (default: `info`)
- `format` — `text` or `json` (default: `text`)

```zsh
z::log::setup "-" info text
z::log::setup "/var/log/app.log" debug
z::log::setup "/var/log/app.log" info json
```

---

### 2. Core Logging

All core functions accept an optional trailing list of key-value pairs.

#### `z::log::error <message> [key val ...]`
Log at level 0 (error). Always reaches console unless logging is fully disabled.

#### `z::log::warn <message> [key val ...]`
Log at level 1 (warning).

#### `z::log::info <message> [key val ...]`
Log at level 2 (info). Default threshold.

#### `z::log::debug <message> [key val ...]`
Log at level 3 (debug). Only emitted when level is set to `debug`.

#### `z::log::log <level> <message> [key val ...]`
Generic log — specify level by name (`error`/`warn`/`info`/`debug`) or number (`0`–`3`).

```zsh
z::log::error "Connection refused" host db.internal retries 3
z::log::info  "Server started" port 8080 env production
z::log::log   warn "Something looks off"
```

---

### 3. Printf-Style Logging

Format a message with `printf` before logging. No key-value pairs supported.

#### `z::log::errorf <format> [args ...]`
#### `z::log::warnf  <format> [args ...]`
#### `z::log::infof  <format> [args ...]`
#### `z::log::debugf <format> [args ...]`

```zsh
z::log::infof  "User %s logged in from %s" "$user" "$ip"
z::log::errorf "Exit code %d from command: %s" "$code" "$cmd"
z::log::debugf "Processing item %d of %d (%.1f%%)" "$i" "$total" "$pct"
z::log::warnf  "Retry %d/%d after %.2fs" "$attempt" "$max" "$delay"
```

---

### 4. Level Guards

Predicate functions that return `0` (true) if the level is currently active. Use to guard expensive operations.

#### `z::log::if_error` — returns `0` if error logging is enabled
#### `z::log::if_warn`  — returns `0` if warn logging is enabled
#### `z::log::if_info`  — returns `0` if info logging is enabled
#### `z::log::if_debug` — returns `0` if debug logging is enabled

```zsh
if z::log::if_debug; then
  local payload=$(serialize_request "$req")
  z::log::debug "Outgoing request" payload "$payload"
fi
```

---

### 5. Control-Flow Helpers

#### `z::log::with_level <level> <command> [args ...]`
Temporarily change the log level for the duration of a command, then restore it. Returns the command's exit code.

#### `z::log::silent <command> [args ...]`
Disable all logging (console and file) for the duration of a command. Returns exit code.

#### `z::log::always <message> [key val ...]`
Force a message through regardless of level. Logged at ERROR level to bypass all filters. Useful for audit trails and startup banners.

```zsh
z::log::with_level debug my_function arg1 arg2
z::log::silent noisy_library_call
z::log::always "Deployment complete" env production sha "$GIT_SHA"
```

---

### 6. Once & Rate Limiting

#### `z::log::once <key> <level> <message> [key val ...]`
Log a message only the first time a given key is seen. Returns `0` always.

#### `z::log::clear_once [key]`
Clear one or all "once" markers, allowing those messages to fire again.

#### `z::log::rate_limit <key> <max_count> <time_window> <level> <message> [key val ...]`
Log at most `max_count` times within `time_window` seconds. Returns `0` if logged, `1` if rate-limited.

#### `z::log::clear_rate_limits [key]`
Clear one or all rate limit counters.

```zsh
z::log::once "deprecated-format" warn "Old log format detected" file "$file"
z::log::rate_limit "disk-warn" 5 60 warn "Disk usage high" usage "$usage%"
z::log::clear_once "deprecated-format"
z::log::clear_rate_limits
```

---

### 7. Context Loggers

Attach a fixed set of key-value pairs to every log call.

#### `z::log::with_context <key1> <val1> [key2 val2 ...]`
Create a context logger. Sets `$REPLY` to the context ID. Even number of args required; keys must match `^[a-zA-Z0-9_-]+$`.

Generated functions (where `$ctx` is the context ID):
- `${ctx}::error <msg> [kv...]`
- `${ctx}::warn  <msg> [kv...]`
- `${ctx}::info  <msg> [kv...]`
- `${ctx}::debug <msg> [kv...]`
- `${ctx}::errorf <fmt> [args...]`
- `${ctx}::warnf  <fmt> [args...]`
- `${ctx}::infof  <fmt> [args...]`
- `${ctx}::debugf <fmt> [args...]`

#### `z::log::remove_context <ctx_id>`
Remove a context logger and undefine its generated functions.

#### `z::log::remove_all_contexts`
Remove all active context loggers.

#### `z::log::list_contexts`
Print all active contexts and their key-value pairs to stdout.

```zsh
z::log::with_context "request_id" "abc-123" "user" "alice"
local ctx="$REPLY"

${ctx}::info  "Request received"
${ctx}::error "Handler failed" code 500
${ctx}::infof "Responded in %dms" "$elapsed"

z::log::remove_context "$ctx"
```

---

### 8. Benchmarking

All benchmarking functions are no-ops when INFO level is disabled, adding zero overhead in production.

#### `z::log::benchmark <name> <command> [args ...]`
Wrap a command and log its execution time. Returns the command's exit code.
Output: `[INFO ] Benchmark completed: <name> | duration=245ms`

#### `z::log::benchmark_start <name>`
Start a named timer. Sets `$REPLY` to the timer ID (empty string if INFO is disabled).

#### `z::log::benchmark_end <timer_id>`
Stop a timer and log the elapsed time. Returns `1` if timer not found.

#### `z::log::benchmark_block <name> <<'END' ... END`
Benchmark a heredoc code block read from stdin.

#### `z::log::benchmark_now`
Get current time in milliseconds. Sets `$REPLY`.

#### `z::log::benchmark_elapsed <start_ms>`
Calculate elapsed time since a start timestamp. Sets `$REPLY` to human-readable duration (e.g., `"1.23s"`).

#### `z::log::list_timers`
Print all active timers with elapsed time.

#### `z::log::clear_timers`
Remove all active benchmark timers without logging results.

```zsh
z::log::benchmark "db_query" run_query --table users --limit 100

z::log::benchmark_start "full_pipeline"
local timer="$REPLY"
step_one && step_two && step_three
z::log::benchmark_end "$timer"

z::log::benchmark_block "data_processing" <<'END'
  for file in /data/*.csv; do process_file "$file"; done
END
```

---

### 9. Timestamp Utilities

#### `z::log::get_timestamp [format]`
Get the current timestamp. Sets `$REPLY`. Formats: `human`/`text` (default), `iso`/`iso8601`, `epoch`/`unix`, `ms`/`milliseconds`, `ns`/`nanoseconds`.

#### `z::log::set_timestamp_format <strftime_format>`
Set a custom `strftime` format for human-readable timestamps (e.g., `"%H:%M:%S"`).

#### `z::log::get_timestamp_format`
Get the current timestamp format string. Sets `$REPLY`.

#### `z::log::reset_timestamp_format`
Reset to default: `%Y-%m-%d %H:%M:%S`.

#### `z::log::format_epoch <epoch> [format]`
Format a Unix epoch timestamp. Sets `$REPLY`.

#### `z::log::time_diff <start_ms> <end_ms>`
Calculate the difference between two millisecond timestamps. Sets `$REPLY` to human-readable duration (`500ms`, `1.50s`, `1m0s`, `1h1m0s`). Returns `1` if `end < start`.

#### `z::log::time_diff_signed <start_ms> <end_ms>`
Like `time_diff` but supports negative durations (e.g., `"-1.50s"`).

#### `z::log::enable_timestamp_cache`
Enable per-second timestamp caching (default). Avoids repeated `strftime` calls.

#### `z::log::disable_timestamp_cache`
Disable caching. Every log call generates a fresh timestamp (~2× slower).

#### `z::log::is_timestamp_cache_enabled`
Returns `0` if caching is enabled, `1` otherwise.

---

### 10. Configuration

#### `z::log::set_level <level>`
Set the console log level. Accepts `error`/`warn`/`info`/`debug` or `0`–`3`.

#### `z::log::get_level`
Get current console log level as a string. Returns via stdout.

#### `z::log::set_file_level <level>`
Set an independent log level for file output. Use `-1` or `console` to follow console level (default).

#### `z::log::get_file_level`
Get the current file log level via stdout.

#### `z::log::set_format <format>`
Set output format: `text` (default) or `json`.

#### `z::log::get_format`
Get current output format via stdout.

#### `z::log::set_file <path>`
Set the log file path. Pass empty string to disable file logging.

#### `z::log::get_file`
Get the current log file path via stdout.

#### `z::log::show_config`
Print a formatted table of all current configuration settings.

#### `z::log::reset`
Reset all configuration to factory defaults (flushes buffers first).

Default values: console level `info`, file level follows console, format `text`, no log file, rotation enabled (10MB, keep 5), buffering disabled, timestamp cache enabled.

#### `z::log::set_max_message_size <size>`
Truncate messages exceeding a size limit. `0` = unlimited (default). Accepts `1024`, `"10KB"`, `"1.5MB"`.

#### `z::log::get_max_message_size`
Get current max message size as human-readable string via stdout.

#### `z::log::set_truncate_marker <marker>`
Set the string appended to truncated messages. Default: ` [TRUNCATED]`.

#### `z::log::get_truncate_marker`
Get the current truncation marker via stdout.

---

### 11. File Rotation

#### `z::log::set_rotation <enabled> [max_size] [keep_count]`
Configure file rotation. `enabled`: `1`/`0`. `max_size`: human-readable (default `10MB`). `keep_count`: int (default `5`).
Rotation scheme: `app.log` → `app.log.1` → … → `app.log.N`. Uses a lock file for multi-process safety.

#### `z::log::set_max_size <bytes>`
Set the maximum log file size in bytes (integers only).

#### `z::log::set_max_files <count>`
Set the number of rotated files to keep.

#### `z::log::set_rotation_lock_timeout <seconds>`
Set the timeout for acquiring the rotation lock file (default: `5` seconds).

```zsh
z::log::set_rotation 1 "50MB" 10
z::log::set_rotation 0   # disable rotation
```

---

### 12. Buffering

#### `z::log::enable_buffering [max_size]`
Enable in-memory buffering. Messages accumulate and are written in bulk. Auto-flush at `max_size` (default 50), on ERROR, on exit, or when `z::log::flush` is called.

#### `z::log::disable_buffering`
Flush pending messages and disable buffering.

#### `z::log::flush`
Flush all buffered messages to the log file in a single bulk write. Safe to call when buffering is disabled (no-op).

#### `z::log::get_buffer_count`
Get the number of messages currently in the buffer via stdout.

#### `z::log::is_buffered`
Returns `0` if buffering is enabled, `1` otherwise.

#### `z::log::set_buffer_size <size>`
Change the auto-flush threshold while buffering is active.

---

### 13. Async Logging (Experimental)

Offloads file I/O to a background worker via a FIFO.

#### `z::log::enable_async`
Start the async logging worker. Requires a log file to be set first. Returns `1` if no log file is configured.

#### `z::log::disable_async`
Gracefully shut down the async worker (waits up to 5 seconds; force-kills after timeout). Safe to call even if async is not enabled.

#### `z::log::is_async`
Returns `0` if async worker is running, `1` otherwise.

---

### 14. Performance Mode

#### `z::log::enable_performance_mode`
Hot-swap the logging engine with a faster implementation (~3–4× speedup). Skips recursion checking, level validation, and error handling. Best combined with `enable_buffering`.

Trade-offs: no recursion protection, no per-call level validation, timestamp cache always on, rotation checks skipped in buffered mode.

#### `z::log::disable_performance_mode`
Restore the original logging engine.

```zsh
z::log::enable_buffering 500
z::log::enable_performance_mode
for i in {1..100000}; do z::log::info "Event $i"; done
z::log::flush
z::log::disable_performance_mode
```

---

### 15. Color System

#### `z::log::colorize <color_spec> <text>`
Apply a color to a string. Sets `$REPLY`. Color specs: `red`, `green`, `blue`, `yellow`, `cyan`, `magenta`, `white`, `black`, `bright_*` variants, semantic (`error`/`warn`/`info`/`debug`/`success`), styles (`bold`/`dim`/`italic`/`underline`), `rgb(R,G,B)` true color, `rgb(R,G,B,bg)` true color background.

#### `z::log::set_color_mode <mode>`
Override auto-detected color mode: `auto` (default), `none`, `basic`, `256`, `truecolor`.

#### `z::log::get_color_mode`
Get current color mode via stdout.

#### `z::log::show_colors`
Print a visual palette of all available colors in the current mode.

---

### 16. Statistics & Diagnostics

#### `z::log::get_stats`
Print runtime statistics (messages logged/dropped, rotations, buffer flushes, errors, active resources).

#### `z::log::reset_stats`
Reset all statistics counters to zero.

#### `z::log::clear_sys_cache [type]`
Clear cached system information. `type`: `hostname`, `username`, `pid`, or `all` (default). Useful after container migration, user switch, or fork.

---

### 17. Cleanup & Resource Management

#### `z::log::cleanup`
Flush buffers, close file descriptors, shut down async worker, remove all contexts, clear timers and rate limits. Idempotent.

#### `z::log::register_cleanup`
Register `z::log::cleanup` to run automatically on script exit (via `add-zsh-hook zshexit` or `TRAPEXIT`). Idempotent.

#### `z::log::unregister_cleanup`
Remove the exit hook registered by `z::log::register_cleanup`.

#### `z::log::get_exit_hook_method`
Get the hook registration method: `"zshexit"`, `"trapexit"`, or `"none"` via stdout.

---

### 18. Internal Debug Mode

Controls zlog's own diagnostic output (separate from application log level). Output goes to stderr.

#### `z::log::enable_debug_mode`
Enable internal zlog diagnostics. Prints `[DEBUG]` lines to stderr for every internal operation.

#### `z::log::disable_debug_mode`
Disable internal diagnostics.

#### `z::log::is_debug_mode`
Returns `0` if internal debug mode is active, `1` otherwise.

---

## Complete Quick Example

```zsh
#!/usr/bin/env zsh
source ./zlog

# Configuration
z::log::setup "/var/log/myapp.log" info text
z::log::set_rotation 1 "50MB" 10
z::log::enable_buffering 100
z::log::register_cleanup

# Basic logging with structured fields
z::log::info  "Application started" version "2.1.0" pid "$$"
z::log::debug "Config loaded"       path "/etc/myapp.conf"
z::log::warn  "Deprecated option"   option "--old-flag"
z::log::error "Connection refused"  host "db.prod" port "5432"

# Printf-style
z::log::infof "Listening on port %d" 8080

# Level guard (avoids expensive call if debug is off)
if z::log::if_debug; then
  z::log::debug "State dump" data "$(serialize_state)"
fi

# Context logger
z::log::with_context "request_id" "req-abc" "user" "alice"
local ctx="$REPLY"
${ctx}::info  "Request received"
${ctx}::error "Handler failed" code 500
z::log::remove_context "$ctx"

# Once & rate limiting
z::log::once       "beta-notice"  info "Beta feature active"
z::log::rate_limit "disk-warn" 5 60 warn "Disk usage high" usage "92%"

# Benchmarking
z::log::benchmark "db_migration" run_migration --env prod

# Control flow
z::log::with_level debug run_verbose_tool
z::log::silent      noisy_library_call
z::log::always "Deployment complete" env production

# Timestamps
z::log::get_timestamp iso
echo "ISO time: $REPLY"
```
